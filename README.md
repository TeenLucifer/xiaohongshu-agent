# 小红书运营多 Agent 系统

## 简述
这是一个面向小红书运营场景的多 Agent 系统，核心目标是把一次运营任务按“话题”管理，并打通从帖子搜集到内容生成的完整链路。

每个话题都是一个独立工作区，单独保存：
- 话题素材
- Agent 会话记录
- 帖子搜集结果
- 人工选择的候选帖子
- 模式总结结果
- 文案生成结果
- 图片生成结果

## 核心流程
系统围绕一个话题按步骤工作：
1. 创建话题并上传素材
2. 运行帖子搜集 Agent
3. 在前端查看所有候选帖子并手动选择、拖动排序
4. 运行模式总结 Agent
5. 运行文案创作 Agent
6. 运行图片创作 Agent

首版以人工逐步触发为主，保证过程可控。

补充约束：
- 帖子搜集 Agent 自主判断停止
- 同时设置最大运行时长和最大候选帖数量作为硬保护

## 功能定义

### 1. 话题与会话
- 以“话题”作为一次运营会话的顶层单位
- 不同话题之间的素材、记录和结果完全隔离
- 同一话题下保留独立的 Agent 会话记录和执行历史
- 同一话题可以反复补充素材、重新搜集、重新生成

### 2. 帖子搜集 Agent
这是系统核心，也是唯一带 loop 的主 Agent。

主要能力：
- 接收话题目标和用户素材作为上下文
- 使用浏览器自动化自主寻找相关高热度帖子
- 默认挂载 file system 和 shell tool
- 支持从 GitHub 下载并安装 skills
- 输出完整的帖子候选集，供用户筛选
- 话题素材中的帖子链接仅作为搜集参考，不直接进入候选帖子区

搜集结果需要尽量完整展示：
- 帖子标题
- 正文内容
- 帖子图片
- 帖子链接
- 作者信息
- 热度相关信息

### 3. 前端候选帖子管理
前端需要完整展示帖子搜集 Agent 找到的所有候选帖子，并把它们作为候选项管理。

首版支持：
- 查看帖子图片和文案
- 查看帖子链接和基础信息
- 手动选择或取消选择候选帖子
- 手动拖动已选候选帖子的顺序

候选帖子区只展示搜集 Agent 实际找到的帖子。
用户选中的候选帖子将作为后续总结和生成的唯一帖子输入。

### 4. 话题素材输入
用户可以为每个话题提供多种素材：
- 图片
- 文本
- 帖子链接

这些素材会被帖子搜集 Agent、文案创作 Agent 和图片创作 Agent 共同参考。

### 5. 模式总结 Agent
这是单轮分析 Agent，不需要 loop。

主要能力：
- 参考用户最终选择的候选帖子
- 总结帖子内容模式和表达规律
- 为后续文案和图片创作提供模式依据

输出方向：
- 标题模式
- 正文结构
- 常见表达方式
- 图片风格特征
- 可复用内容模板

### 6. 文案创作 Agent
这是单轮生成 Agent，不需要 loop。

主要能力：
- 根据用户提供的素材
- 参考用户选中的候选帖子
- 结合模式总结 Agent 的输出
- 生成文案候选结果

输出方向：
- 完整笔记草稿
- 标题
- 正文
- 基础内容结构

### 7. 图片创作 Agent
这是单轮生成 Agent，不需要 loop，但承担两类图片创作任务。

功能一：文本生图
- 根据用户素材、已绑定的文案草稿和模式总结生成图片

功能二：图生图
- 根据用户素材图片
- 参考搜集到的帖子图片
- 结合用户文本描述和模式总结
- 生成新的图片结果

图片创作 Agent 的目标是直接产出候选图片，而不只是生成提示词。
首版图片结果以封面和内页候选图为主，每次出图都绑定一版明确的文案草稿。

## 前后端职责

### 前端
- 创建话题
- 上传素材
- 查看 Agent 会话记录
- 展示候选帖子
- 完成候选帖子选择和排序
- 查看总结结果、文案结果和图片结果
- 会话记录默认展示输入输出摘要和状态，并支持展开查看详细日志

### 后端
- 负责任务编排和 Agent 调度
- 管理话题、素材、会话记录和结果数据
- 管理 skills 安装与加载
- 管理浏览器会话和执行环境

## 预期效果
- 用户可以围绕一个话题持续积累素材和结果，而不是一次性跑完即结束
- 系统可以自动搜集相关高热度帖子，减少人工找样本的成本
- 用户可以先人工筛选候选帖子，再让系统基于这些帖子做总结和生成
- 文案和图片生成会同时参考用户素材、候选帖子和模式总结，结果更贴近运营目标

## 首版边界
- 首版是单用户工作台，不先做团队协作
- 首版以人工逐步触发流程为主，不做全自动串行
- 小红书搜集依赖浏览器自动化和登录态复用
- skills 首版只服务帖子搜集 Agent
