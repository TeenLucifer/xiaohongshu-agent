# 005 Home Entry

## 背景

当前前端虽然已经完成了 topic 列表、创建、删除与工作台路由，但 `/` 仍然更像一个“管理页”，而不是一个明确的产品首页入口。页面中心存在多字段表单和较重的信息块，不符合“小红书运营台”作为创作型产品的起始心智。

同时，工作台左侧导航也缺少一个清晰的“返回起点”入口。用户进入某个 topic 后，应该能随时回到一个干净的起始页，新建下一个话题，或继续最近的话题工作。

## 目标

- 将 `/` 定义为正式首页，名称为“新话题”
- 让首页视觉中心只保留一个主入口：直接输入标题创建话题
- 让历史话题作为次入口出现在首页下方，而不是与主入口并列竞争
- 让首页整体参考 ChatGPT 风格的极简起始页，而不是后台管理页
- 在工作台左侧控制栏中增加“新话题”入口，并将其放在导航列表第一项
- 点击左侧“新话题”时返回首页
- 首页继续复用现有 topic API，不新增后端接口

## 非目标

- 不重做 topic 管理后端能力
- 不在首页引入描述输入、模板推荐、统计卡片或 dashboard 区
- 不调整 topic 创建后的工作台结构
- 不在本轮处理首页之外的其它产品信息架构

## 用户故事

- 作为运营人员，我希望打开产品后看到一个干净明确的“新话题”入口，而不是抽象的管理页面。
- 作为运营人员，我希望像 ChatGPT 首页那样，直接输入一句标题就能开始新的 topic 工作。
- 作为运营人员，我希望在首页下方继续最近的话题，而不用重新搜索或理解后台结构。
- 作为运营人员，我希望在工作台内部随时通过左侧“新话题”回到首页开始新的工作。

## 输入输出

- 输入：真实 topic 列表、topic 创建接口
- 输出：首页“新话题”入口与历史话题次入口

## 约束

- 前端代码放在 `web/`
- 首页主入口采用单输入框形式
- 第一版创建时只输入标题，不输入描述
- 历史话题采用简洁列表，不使用卡片墙
- 历史话题列表允许保留最小进入与删除操作，但整体视觉权重仍低于中央主创建入口
- 首页继续复用：
  - `GET /api/topics`
  - `POST /api/topics`
- 工作台左侧栏中的“新话题”是一级真实入口，不是占位按钮
- 首页与工作台都继续走现有路由：
  - `/`
  - `/topics/:topicId`
- 创建成功后直接跳转到 `/topics/{topic_id}`

## 验收标准

- 打开前端应用后默认进入“新话题”首页
- 首页中央能直接输入标题并创建 topic
- 创建成功后会跳转到对应 topic 工作台
- 首页下方展示历史话题列表，且列表作为次入口存在感低于主创建入口
- 历史话题列表点击后可进入对应 topic
- 历史话题列表可包含删除操作，但不应改变首页“新话题”作为主入口的产品心智
- 没有历史话题时首页显示极简空状态
- 工作台左侧导航第一项为“新话题”
- 点击左侧“新话题”后可返回首页
- 首页不再表现为后台式管理页
