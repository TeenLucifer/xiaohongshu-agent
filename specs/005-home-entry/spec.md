# 005 Home Entry

## 背景

当前前端虽然已经完成了 topic 列表、创建、删除与工作台路由，但 `/` 仍然更像一个“管理页”，而不是一个明确的产品首页入口。页面中心存在多字段表单和较重的信息块，不符合“小红书运营台”作为创作型产品的起始心智。

同时，工作台左侧导航也缺少一个清晰的“返回起点”入口。用户进入某个 topic 后，应该能随时回到一个干净的起始页，新建下一个话题，或继续最近的话题工作。

## 目标

- 将 `/` 定义为正式首页，名称为“新话题”
- 让首页视觉中心只保留一个主入口：直接输入标题创建话题
- 让首页主入口采用居中的单行胶囊输入框，接近“项目命名框”而不是聊天输入框
- 让首页主入口成为首屏唯一主焦点，通过结构主次而不是堆叠更多卡片建立注意力
- 让首页主入口的色系与当前工作区保持一致，采用白 / 灰 / 蓝的冷静体系，而不是单独的暖色高光或过重渐变
- 让近期话题作为次入口收成页面底部的一条横向细轨道，而不是纵向列表
- 让近期话题卡片采用 3:4 的竖卡比例，视觉节奏接近候选帖子卡片
- 让近期话题卡片内部不显示冗余的“最近话题”文案
- 让近期话题卡片在已有生成图时，优先使用第一张生成图作为背景图
- 让首页整体更接近“中心主入口 + 辅助继续入口”的产品首页，而不是后台管理页
- 在工作台左侧控制栏中增加“新话题”入口，并将其放在导航列表第一项
- 点击左侧“新话题”时返回首页
- 首页继续复用现有 topic API，不新增后端接口

## 非目标

- 不重做 topic 管理后端能力
- 不在首页引入描述输入、模板推荐、统计卡片或 dashboard 区
- 不调整 topic 创建后的工作台结构
- 不在本轮处理首页之外的其它产品信息架构

## 用户故事

- 作为运营人员，我希望打开产品后看到一个干净明确的“新话题”入口，而不是抽象的管理页面。
- 作为运营人员，我希望首页中央只有一个明确的创建入口，直接输入一句标题就能开始新的 topic 工作。
- 作为运营人员，我希望首页底部能轻量地继续最近的话题，而不是面对一整块历史管理列表。
- 作为运营人员，我希望在工作台内部随时通过左侧“新话题”回到首页开始新的工作。

## 输入输出

- 输入：真实 topic 列表、topic 创建接口
- 输出：首页“新话题”入口与历史话题次入口

## 约束

- 前端代码放在 `web/`
- 首页主入口采用单输入框形式
- 首页主入口采用单行胶囊输入框，不使用多行 `textarea`
- 首页主入口默认视觉应更接近单行命名框，而不是宽大的聊天输入框
- 首页主入口所在区域应作为首页唯一主焦点，且色系需与工作区的白 / 灰 / 蓝视觉语言一致
- 近期话题必须收成横向单行滚动细轨道，不得再使用纵向大列表
- 近期话题卡片采用 3:4 竖卡，不要求真实封面图，但应通过抽象封面区避免纯文字卡片过于单调
- 近期话题卡片内部不显示“最近话题”这类重复标签
- 若话题已有生成图，则首页卡片应优先展示第一张生成图并占满整张卡片；无图时才回退抽象背景
- 第一版创建时只输入标题，不输入描述
- 近期话题允许保留最小进入与删除操作，但整体视觉权重仍低于中央主创建入口
- 近期话题次信息使用相对时间，不直接展示后端原始时间戳
- 首页继续复用：
  - `GET /api/topics`
  - `POST /api/topics`
- 工作台左侧栏中的“新话题”是一级真实入口，不是占位按钮
- 首页与工作台都继续走现有路由：
  - `/`
  - `/topics/:topicId`
- 创建成功后直接跳转到 `/topics/{topic_id}`

## 验收标准

- 打开前端应用后默认进入“新话题”首页
- 首页中央能直接输入标题并创建 topic
- 首页中央创建区应为单行胶囊输入框，视觉上接近“项目命名框”
- 首页中央创建区的色系应与当前工作区保持一致，不应显得像另一套落地页
- 首页中央创建区应明显先于“近期话题”被感知，近期话题为次入口
- 创建成功后会跳转到对应 topic 工作台
- 首页底部展示横向单行滚动的近期话题轨道
- 近期话题卡片应采用 3:4 竖卡比例，并通过抽象封面区承载标题
- 近期话题卡片内部不应再显示“最近话题”字样
- 已生成过图片的话题，应优先显示第一张生成图并占满整张卡片
- 近期话题卡片点击后可进入对应 topic
- 近期话题卡片次信息应显示相对时间，而不是绝对时间戳
- 近期话题卡片可包含删除操作，但不应改变首页“新话题”作为主入口的产品心智
- 没有历史话题时首页显示极简空状态
- 工作台左侧导航第一项为“新话题”
- 点击左侧“新话题”后可返回首页
- 首页不再表现为后台式管理页
