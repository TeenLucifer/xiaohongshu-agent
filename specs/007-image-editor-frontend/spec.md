# 007 Image Editor Frontend

## 背景

当前右侧"图片"区域位于"创作" tab 中，仅展示生成结果，缺少对搜索结果图片的管理能力。运营人员需要：
- 查看所有搜索结果的图片
- 选择感兴趣的图片进入编辑区
- 通过编号在对话中引用图片

## 目标

- 在"创作" tab 的"图片"区域实现上下分栏布局
- 上半部分展示素材图片（搜索结果的所有图片）
- 下半部分展示编辑区图片（用户选择的图片，带编号）
- 支持拖拽添加、点击移除
- 编号全局连续，移除后自动重排
- 编辑区图片状态持久化到 `editor_images.json`
- 支持接收生成结果区的图片拖入编辑区

## 非目标

- 图片编辑功能（裁剪、滤镜等）
- 图片上传功能
- 多话题图片混用
- 图片真实生成本身

## 用户故事

- 作为运营人员，我希望在搜索帖子后，能在"创作" tab 的图片区域看到所有候选帖子的图片。
- 作为运营人员，我希望能把想要的图片拖到编辑区，并获得一个编号。
- 作为运营人员，我希望能通过编号在对话中引用图片。

## 交互规则

### 素材区

- 展示所有候选帖子的图片，按帖子分组显示
- 图片按帖子顺序排列，同一帖子内按图片顺序排列
- 图片可拖拽到编辑区

### 编辑区

- 拖入的图片自动获得编号（1, 2, 3...）
- 编号显示在图片左上角
- 点击图片可移除
- 移除后编号自动重排
- 编辑区图片既可来自搜索素材，也可来自生成结果
- 生成结果图进入编辑区的方式与素材图一致：拖拽进入，不提供单独“加入”按钮

### 拖拽行为

- 使用 HTML5 Drag & Drop API
- 拖拽时显示半透明预览
- 拖入编辑区时高亮显示目标区域
- 同一张图片可多次拖入

## 数据结构

```typescript
// 素材图片
interface MaterialImage {
  id: string;           // 唯一 ID
  postId: string;       // 来源帖子 ID
  postTitle: string;    // 来源帖子标题
  imageUrl: string;     // 图片 URL
  alt: string;          // 描述
}

// 编辑区图片
interface EditorImage {
  id: string;            // 唯一 ID
  order: number;         // 编号 1, 2, 3...
  sourcePostId: string;  // 来源帖子 ID
  sourceImageId: string; // 来源图片 ID
  imageUrl: string;      // 图片 URL
  alt: string;           // 描述
}
```

## 验收标准

- [ ] 图片区域位于右侧工作区"创作" tab
- [ ] 素材区显示所有候选帖子的图片
- [ ] 拖拽图片到编辑区，显示编号
- [ ] 移除图片后编号重排
- [ ] 编辑区图片状态持久化到后端
- [ ] 可从生成结果区拖拽图片到编辑区
