# 029 Settings Page Acceptance

## 手工验收

1. 打开前端应用
2. 点击侧边栏“设置”
3. 确认进入 `/settings`
4. 确认页面首屏展示：
   - 短标题区
   - 顶部 tabs
   - 单一主内容面板
5. 确认 tabs 分别为：
   - 主 LLM
   - 图片识别
   - 图片生成
6. 确认默认激活“主 LLM”
7. 确认每次只展示一个激活 tab 对应的主面板
8. 确认主面板都有：
   - Base URL
   - API Key
   - Model
   - 保存按钮
   - 测试按钮
9. 确认切换 tabs 后，下方主面板内容随之切换
10. 确认 API Key 默认以普通字符节奏的星号掩码回填当前值
11. 点击小眼睛后，确认可显示真实 API Key
12. 在主 LLM 面板中修改 `base_url / api_key / model` 并保存
13. 确认保存成功后，后续普通对话请求直接使用新配置
14. 在图片识别面板中修改配置并保存
15. 确认图片识别相关能力后续调用使用新配置
16. 在图片生成面板中修改配置并保存
17. 确认图片生成后续调用使用新配置
18. 在三个 tab 中分别点击“测试”
19. 确认每个面板都能返回清晰的成功/失败结果提示
20. 输入新 API Key 后保存，确认旧 key 会被覆盖
21. 不输入新 API Key 时保存，确认旧 key 会被保留
22. 确认设置页整体风格与当前工作区一致，但层级更接近独立设置页

## 自动化验收

- `/settings` 路由渲染测试通过
- `GET /api/settings` 读取测试通过
- 主 LLM 配置写回与 runtime 热刷新测试通过
- 图片识别/图片生成配置写回测试通过
- 三类连接测试接口的成功/失败测试通过
- 侧边栏“设置”导航测试通过

## 已知限制

- 第一版只支持 `base_url / api_key / model`
- 图片生成的 `size` 暂不进入设置页 UI
- 第一版不做主题、语言、本地缓存等通用设置
