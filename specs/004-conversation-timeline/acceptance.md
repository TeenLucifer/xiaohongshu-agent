# 004 Conversation Timeline Acceptance

## 手工验收

1. 进入工作区主面板并切到 `对话` tab
2. 确认 `对话` tab 中看到的是 user / agent 的简化对话流，而不是日志时间线
3. 查看任意一条 user 消息和一条 agent 回复消息
4. 确认主界面看不到调用过程、工具过程、输入摘要、输出摘要、详细日志和结果摘要块
5. 准备一轮包含多个 tool 调用的 agent 执行
6. 确认主栏只展示该轮 final answer，不展示中间 assistant 迭代文本
7. 确认该 final answer 下方可看到一个默认折叠的工具调用摘要
8. 展开工具调用摘要，确认其中只包含轻量的工具名称、参数摘要和结果摘要
9. 确认每条消息都采用较小的图标 + 文本结构
10. 确认总结结果和文案结果都能以 agent 回复消息形式出现
11. 准备一条包含标题、列表、引用、代码块、链接或表格的 agent 消息
12. 确认这些 Markdown 结构能正确显示，而不是退化成纯文本
13. 确认 user 消息仍按普通文本显示，不自动解释 Markdown
14. 确认代码块和表格不会撑爆消息气泡
15. 确认对话消息字号层级清晰，适合快速扫读
16. 确认 `对话` tab 中的顶部说明和输入区不会压过对话主体
17. 准备一轮图片生成结果，确认 final answer 下可附带 1 张代表缩略图
18. 点击该缩略图，确认可进入统一图片预览
19. 在 `创作` tab 的文案区执行一次选区 AI 润色，确认 `对话` tab 中新增一条结果消息
20. 在 `对话` tab 中发送一条新消息，确认对话区会自动滚动到底部
21. 在 `创作` tab 顶部的搜索结果局部输入框发送一条消息，确认 `对话` tab 中也能看到该记录
22. 在 `创作` tab 的图片区局部输入框发送一条消息，确认 `对话` tab 中也能看到该记录

- 中间 assistant 迭代过滤测试通过

- final answer 图片附件渲染测试通过
- 选区 AI 润色结果消息测试通过
- 新消息自动贴底测试通过
