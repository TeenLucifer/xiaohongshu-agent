# 004 Conversation Timeline Tasks

## 开发子步骤

- [x] `004-A`：重定义对话节点展示模型
  覆盖会话节点、内容节点、状态字段和详细日志内容。
- [x] `004-B`：实现中间主栏对话时间线
  展示时间、状态、输入摘要和输出摘要。
- [x] `004-C`：实现单条节点展开日志
  让用户按需查看详细日志。
- [x] `004-D`：统一中间主栏消息类型
  让会话节点、文案节点、总结节点能共存。
- [x] `004-E`：统一时间线字体层级和状态标签样式
  保持摘要态易扫读，状态区分清晰但克制。
- [x] `004-F`：优化展开日志过渡
  避免瞬时开关，统一到轻量过渡节奏。
- [x] `004-G`：将时间线重构为简化对话流
  主界面只保留 user 消息和 agent 结果消息。
- [x] `004-H`：移除主界面过程信息
  不再默认展示调用过程、工具过程、输入输出摘要和详细日志。
- [x] `004-I`：将对话消息压缩为轻量行式结构
  采用图标 + 文本的小体量消息样式，参考 DeepTutor。
- [x] `004-J`：移除主界面的结果摘要块
  主栏只保留 user / agent 文字消息，不再额外插入结果摘要面板。
- [x] `004-K`：使用 Lucide 统一主栏结构型图标
  图标只用于消息角色和结构引导，不进入正文内容装饰。
- [x] `004-L`：将主栏容器切换到 Tailwind + Framer Motion 表达
  统一主栏布局、消息容器和页面级动效形式。
