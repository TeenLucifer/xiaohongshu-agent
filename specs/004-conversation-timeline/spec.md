# 004 Conversation Timeline

## 背景

在新版 workspace-first 工作台里，对话不再常驻在页面中间，而是被收进工作区的 `对话` tab。该 tab 继续负责展示 user / agent 消息流、final answer、工具摘要和图片附件，但不再承担整个页面唯一主焦点的职责。

## 目标

- 让 `对话` tab 以简化对话方式展示 user 与 agent 的消息
- 让每条消息保持 DeepTutor 风格的小体量展示
- 让 `对话` tab 只保留结果信息，不展示过程性记录
- 让 agent 消息支持 GFM 基础 Markdown 显示，提升可读性
- 让 final answer 支持附带轻量图片附件结果
- 让每次发送新消息或收到新回复时，对话区自动滚动到底部

## 非目标

- 实时日志流
- 日志搜索和筛选
- 真实日志来源联调
- 富文本消息编辑器
- 原始 HTML 渲染
- 代码高亮
- 在主界面展示调用过程、工具过程或详细执行日志

## 用户故事

- 作为运营人员，我希望中间主栏本身就是可读的 user / agent 对话，而不是日志时间线。
- 作为运营人员，我希望默认只看结果，不想被调用过程和工具细节打断。
- 作为运营人员，我希望 agent 输出的列表、标题、代码块和链接能按 Markdown 正确显示，而不是退化成一大段纯文本。
- 作为开发者，我希望先固定简化对话结构，后续只替换真实会话数据。

## 输入输出

- 输入：Mock user 消息、Mock agent 结果消息
- 输出：中间主栏的简化对话流

## 约束

- `对话` tab 只展示两类消息
  - user 发送的文字
  - agent 返回的结果文字
- 主界面不展示调用过程、工具过程、输入摘要、输出摘要、详细日志、结果摘要块
- 如未来需要过程信息，只能进入独立 debug 视图，不在主界面默认展示
- agent 消息只展示每个 user turn 的 final answer
- loop 过程中产生的中间 assistant 文本不进入主栏
- tool 调用过程不以独立消息展示，但可作为附着在 final answer 下方的折叠摘要存在
- 当某个 final answer 产出了图片结果时，可在同一条回复下附带轻量图片附件
- 当右侧文案区执行选区 AI 润色时，中间对话流应追加一条结果消息，用于记录本次润色执行结果
- 当 `创作` tab 内的局部输入框发送消息时，中间对话流也应追加对应的 user / agent 记录
- 第一版每条 final answer 最多展示 1 张代表图缩略结果
- 每条消息需要采用小体量对话样式
  - 一行消息容器 + 一个图标 + 文本内容
  - 整体体量参考 DeepTutor
- 中间主栏需兼容总结结果、文案结果等不同 agent 回复内容，但都按同一对话样式展示
- agent 消息正文支持 GFM 基础 Markdown：
  - 标题
  - 段落
  - 列表
  - 引用
  - 代码块
  - 行内代码
  - 链接
  - 表格
  - 粗体 / 斜体
- user 消息继续按普通文本显示，不自动解释 Markdown
- Markdown 渲染不支持原始 HTML
- 代码块与表格在窄容器内需要可读，不允许撑爆消息气泡
- 字号和层级需要统一，适合快速扫读
- 对话消息不使用高噪声状态标签
- `对话` tab 内需要保持对话内容为主，输入框仅在该 tab 中出现
- 主输入框与局部输入框都需要支持长文本自动换行
  - 默认自动增高到上限
  - 超过上限后在输入框内部滚动
  - `Enter` 发送，`Shift + Enter` 换行
- `创作` 中的局部输入动作与 `对话` tab 主输入框共享同一条消息流真相
- `对话` tab 不保留额外说明头
  - 不显示 `Conversation`
  - 不显示 `对话记录`
  - 不显示执行反馈说明
  - 不显示消息数 badge
- 当新增 user 消息、streaming agent 消息或最终回复落入列表时，对话滚动容器应自动贴底

## 验收标准

- `对话` tab 按 user / agent 简化对话流展示内容
- 主界面看不到调用过程、工具过程、输入摘要、输出摘要和详细日志
- 若某个 final answer 有工具调用摘要，默认以折叠形式附着在该回复下方
- 若某个 final answer 有图片结果，默认可在回复下方看到代表图缩略结果
- 每条对话消息体量较小，采用图标 + 文本的轻量结构
- 总结结果和文案结果都能以 agent 回复消息形式出现
- agent 消息中的 Markdown 结构可正确显示
- user 消息仍保持普通文本回显
- 对话消息整体易于扫读
- 进入 `对话` tab 后，消息流保持该 tab 内的主要视觉内容
- `对话` tab 内不再出现说明性抬头和统计文案
- 新消息和新回复到达时，对话区会自动滚动到底部
- 在其它 tab 触发的局部输入消息，切到 `对话` tab 后也能看到统一记录
- 主输入框和局部输入框在长文本场景下都能自动换行和增高
