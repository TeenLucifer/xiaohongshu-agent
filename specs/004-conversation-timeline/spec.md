# 004 Conversation Timeline

## 背景

在新版工作台里，中间主栏需要从“时间线/过程日志”重构为“简化对话流”。主界面只展示 user 发送的文字和 agent 返回的结果文字，不展示调用过程、工具过程和执行日志。过程信息如果未来要保留，也只能作为可选 debug 视图存在，不进入主界面。

## 目标

- 让中间主栏以简化对话方式展示 user 与 agent 的消息
- 让每条消息保持 DeepTutor 风格的小体量展示
- 让中间主栏只保留结果信息，不展示过程性记录

## 非目标

- 实时日志流
- 日志搜索和筛选
- 真实日志来源联调
- 富文本消息编辑器
- 在主界面展示调用过程、工具过程或详细执行日志

## 用户故事

- 作为运营人员，我希望中间主栏本身就是可读的 user / agent 对话，而不是日志时间线。
- 作为运营人员，我希望默认只看结果，不想被调用过程和工具细节打断。
- 作为开发者，我希望先固定简化对话结构，后续只替换真实会话数据。

## 输入输出

- 输入：Mock user 消息、Mock agent 结果消息
- 输出：中间主栏的简化对话流

## 约束

- 主界面只展示两类消息
  - user 发送的文字
  - agent 返回的结果文字
- 主界面不展示调用过程、工具过程、输入摘要、输出摘要、详细日志、结果摘要块
- 如未来需要过程信息，只能进入独立 debug 视图，不在主界面默认展示
- 每条消息需要采用小体量对话样式
  - 一行消息容器 + 一个图标 + 文本内容
  - 整体体量参考 DeepTutor
- 中间主栏需兼容总结结果、文案结果等不同 agent 回复内容，但都按同一对话样式展示
- 字号和层级需要统一，适合快速扫读
- 对话消息不使用高噪声状态标签
- 对话区需要成为中栏绝对主内容
  - 顶部说明和输入区都不能压过对话主体

## 验收标准

- 中间主栏按 user / agent 简化对话流展示内容
- 主界面看不到调用过程、工具过程、输入摘要、输出摘要和详细日志
- 每条对话消息体量较小，采用图标 + 文本的轻量结构
- 总结结果和文案结果都能以 agent 回复消息形式出现
- 对话消息整体易于扫读
- 对话区在中栏中保持最强视觉优先级
