# 028 Workspace Image Uploads

## 背景

当前工作区已经能承接帖子、总结、文案和图片，但独立“素材区”的心智过重，和真实使用路径不一致。用户真正高频需要补充的是**本地图片**，而不是单独维护一块文本/链接/图片混合素材池。

因此这条能力收口为：直接在 `创作 -> 图片` 区里上传本地图片，并把它们作为候选图与帖子图一起使用。

## 目标

- 移除 `选题` tab 中独立的“素材”section
- 在 `创作 -> 图片` section 内提供本地图片上传入口
- 让上传图片写入当前 session workspace
- 让上传图片自动进入图片区候选列表，并排在帖子图之前
- 继续复用现有 `materials.json` / `materials/` 作为图片上传底层存储

## 非目标

- 文本素材与链接素材的产品化入口
- 独立素材池
- 图片裁剪、重命名或复杂批处理
- 批量删除、批量选择

## 用户故事

- 作为运营人员，我希望在图片区直接上传本地图片，而不是先去一个单独素材区。
- 作为运营人员，我希望上传的图片能立刻出现在图片区候选列表里，继续拖拽到编辑区使用。
- 作为运营人员，我希望删除上传图片时前后端同步移除，不留下假数据。

## 输入输出

- 输入：
  - 图片文件
- 输出：
  - `workspace/materials.json`
  - `workspace/materials/` 下的图片文件
  - `/context` 中真实图片上传记录
  - `创作` tab 图片候选列表中的上传图片

## 约束

- 不再渲染 `选题` tab 的独立素材 section
- 图片上传入口放在 `创作 -> 图片` section 的候选图片区内部
- 上传入口支持点击上传，并支持把本地图片文件直接拖到上传入口
- 图片支持一次上传多张
- 上传图片进入图片区候选列表，顺序固定为：上传图片优先，帖子图其次
- 上传图片卡 hover 右上角显示删除 `X`
- 上传图片与帖子图都可点击预览，并继续拖入编辑区
- 两类候选图都可拖入编辑区
- 删除上传图片后：
  - 图片上传记录被移除
  - 若编辑区引用了该图片，则相应编辑区图片也一并移除
- 第一版不做文本素材与链接素材入口

## 真相层

- `workspace/materials.json`
- `workspace/materials/<asset files>`

`materials.json` 继续复用，但当前产品主路径只要求保存图片上传记录。记录至少包含：

- `id`
- `type: image`
- `title`
- `image_path`
- `created_at`

## 后端接口

- `GET /api/topics/{topic_id}/materials`
- `POST /api/topics/{topic_id}/materials/images`
- `DELETE /api/topics/{topic_id}/materials/{material_id}`

同时：

- `GET /api/topics/{topic_id}/context` 需要返回真实 `materials`

## 验收标准

- 页面中不再出现独立“素材”section
- 用户可以在 `创作 -> 图片` 区直接上传本地图片
- 图片支持批量上传
- 上传入口支持点击上传，且可作为本地拖入上传入口
- 刷新页面后上传图片仍能从 workspace 恢复
- 删除上传图片后，前后端状态同步移除
- 上传图片会出现在 `创作` tab 的图片区候选列表中
- 候选顺序为：上传图片优先，帖子图其次
- 两类候选图都可预览并拖入编辑区
- 上传图片默认成为 agent 与图片区可参考输入，不需要额外“加入”动作
