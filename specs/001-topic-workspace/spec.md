# 001 Topic Workspace

## 背景

工作台已经从早期“三栏里中栏对话为主、右栏工作区为辅”的形态，演进为更贴近真实使用方式的 **workspace-first** 布局。当前用户的主要操作都发生在工作区内部，因此页面需要改成：

- 左侧保留导航
- 主内容区改为一个铺满页面的大工作区
- 主工作区收口为 `创作 / 对话` 两个 tab
- 原来的对话主栏收进工作区，作为单独的 `对话` tab

## 目标

- 建立“左侧导航 + 单一工作区主面板”的桌面布局
- 让主内容区通过 `创作 / 对话` 两个 tab 承接所有工作流
- 让 `对话` 成为工作区中的结果记录 tab，而不是页面唯一主焦点
- 让输入框只在 `对话` tab 中出现
- 让 `创作` 中触发 agent 动作后停留在当前 tab，不自动跳转到 `对话`
- 保留当前视觉语言、滚动模型与轻量动效

## 非目标

- Agent 执行逻辑
- 新的运行协议
- 复杂协作能力
- 完整视觉像素级复刻参考产品

## 用户故事

- 作为运营人员，我希望打开话题后首先看到的是完整工作区，而不是“中间聊天、右侧辅助栏”的拆分布局。
- 作为运营人员，我希望搜索结果、总结、文案和图片都在同一个大工作区里切换查看。
- 作为运营人员，我希望对话更像结果记录和执行历史，需要时再进入查看，而不是一直占据页面中心。

## 输入输出

- 输入：
  - 真实 topic 列表
  - 真实 `/messages`、`/context`、`/runs`、`/runs/stream`
  - 已有的工作区对象存储与图片/文案/总结能力
- 输出：以 tab 组织的单一大工作区页面

## 约束

- 前端代码放在 `web/`
- 继续使用：
  - `Tailwind CSS`
  - 轻量基础组件体系
  - `Lucide`
  - `Framer Motion`
- 页面结构固定为两列：
  - 左侧导航栏
  - 右侧工作区主面板
- 左侧导航栏继续支持收起和展开
  - 默认展开
  - 收起后缩到 `72-88px`
  - 继续贴住页面左侧、顶部和底部
  - 继续不显示无真实承载价值的占位导航项
- 主工作区面板铺满剩余宽度，不再拆分为中栏和右栏
- 主工作区顶部收口为轻量标题栏
  - 仅保留话题标题与删除按钮
  - 不保留 `Topic Workspace`、话题描述或状态 badge
- 主工作区顶部提供 tab：
  - `创作`
  - `对话`
  - 默认显示 `创作`
- `创作` tab 承接：
  - 搜索结果（内含总结折叠块与局部输入框）
  - 文案
  - 图片（内含局部输入框）
- `对话` tab 承接：
  - user / agent 消息流
  - final answer
  - tool 折叠摘要
  - 图片附件
  - 流式输出
- 输入框只在 `对话` tab 中显示
- 在 `创作` tab 中点击触发 run 的按钮后：
  - 保持停留在当前 tab
  - 当前 tab 继续展示 loading / success / error
  - `对话` tab 记录本轮执行结果
- `创作` 中的局部输入框也复用同一条 run 链路
  - 发送后保持停留在当前 tab
  - 对话记录统一进入 `对话` tab
- 各 section 头部不显示 `已就绪` 或 `进行中` 状态 badge
- `对话` tab 采用极简结构
  - 不保留额外说明头
  - 不显示 `Conversation`、`对话记录`、执行反馈说明或消息数 badge
  - 以消息流和输入框为主
- 页面继续采用 `100vh` 内部滚动模型
  - 左侧导航独立滚动
  - 主工作区独立滚动
- 保持浅色主题、统一字号层级、弱边框、轻阴影、大圆角和轻量容器级动效

## 验收标准

- 进入任意 topic 后显示“左侧导航 + 单一工作区主面板”
- 主工作区显示 `创作 / 对话` 两个 tab
- 默认 tab 为 `创作`
- 左侧导航收起后，主工作区直接扩展占满剩余宽度
- 页面中不再存在独立的右侧工作区收起/展开交互
- `创作` tab 中最上方能看到搜索结果，且总结作为搜索结果区下方的一部分呈现
- `创作` tab 中在搜索结果下方能看到文案、图片，且图片区内带局部输入框
- `对话` tab 中能看到消息流，并且输入框只在该 tab 中出现
- 页面中不再出现 `Topic Workspace`、话题描述、`已就绪`
- 在 `创作` 中触发生成动作后，页面保持停留在当前 tab
- 切换到 `对话` tab 后能看到对应的 user / agent 结果记录
- 在 `创作` 的局部输入框发送消息后，当前 tab 不切走，且 `对话` tab 可看到同一轮记录
- 页面视觉仍然保持当前工作台的轻量风格和内部滚动体验
