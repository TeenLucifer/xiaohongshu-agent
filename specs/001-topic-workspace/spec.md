# 001 Topic Workspace

## 背景

前端需要从当前的纵向长页面重构为参考图风格的三栏工作台。参考来源以 `assests/references/ScreenShot_2026-03-26_201829_887.png`、`localhost:3782` 和本机 `DeepTutor` 项目的前端设计语言为准。

## 目标

- 建立进入话题后的三栏工作台结构
- 建立左侧导航栏、中间对话主栏、右侧工作区三栏结构
- 中间主栏最终定为“聊天产品式主栏”
- 让左侧导航栏支持收起和展开
- 让右侧工作区支持收起和展开
- 让右侧工作区收起时，中间对话区同步扩张
- 固定新版工作台的布局、层级和基础状态
- 固定全局基础视觉语言，包括字号层级、圆角、阴影、边框和基础动效
- 让中间主栏成为唯一主焦点，右侧工作区退为辅助上下文区

## 非目标

- Agent 执行逻辑
- 复杂权限或协作能力
- 完整视觉像素级复刻参考产品

## 用户故事

- 作为运营人员，我希望在进入某个话题后看到稳定的三栏工作台，而不是纵向长页面。
- 作为运营人员，我希望中间是清晰的对话主栏，右边是当前工作区，整体更接近成熟 Agent 产品。
- 作为开发者，我希望先固定新版工作台结构，再在这个结构上继续补后续交互。

## 输入输出

- 输入：
  - 真实 topic 列表与主栏会话 API
  - 右侧 `candidatePosts` / `patternSummary` 真实读取 API
  - 其余右侧区块可继续使用 mock 数据
- 输出：可浏览的话题工作台页面

## 约束

- 前端代码放在 `web/`
- 前端视觉实现基线采用：
  - `Tailwind CSS`
  - `自定义轻量基础组件体系`
  - `Lucide` 图标
  - `Framer Motion`
- Tailwind 使用方式采用 `utility-first + 少量基础样式`
  - 大部分页面样式直接使用 Tailwind utilities
  - 全局只保留 reset、token 和极少量公共规则
- 不引入重型 UI 组件库
  - 不采用 `Ant Design`
  - 不采用 `MUI`
  - 不采用 `Chakra`
- 自定义组件体系只覆盖轻量基础件
  - Button
  - Input / ChatInput
  - Badge / Tag
  - Panel / Surface
  - Modal
  - ScrollArea
  - SidebarItem
- 基础组件自己管理 `variant / size / state`
- 允许引入轻量样式辅助
  - `clsx`
  - `tailwind-merge`
  - 必要时可加 `cva`
- Lucide 只用于结构型位置
  - 左侧导航
  - 输入框发送按钮
  - 收起/展开按钮
  - 弹窗关闭按钮
  - 少量状态标记
- Framer Motion 只用于页面级和容器级动效
  - 左侧栏收起/展开
  - 右侧工作区收起/展开
  - 分组开合
  - 弹窗进入/退出
  - 页面首屏淡入
- 本轮只做浅色主题
- 保留中文安全系统字体栈，只调整字号、字重、字距和层级
- 本 feature 只定义进入话题后的工作台，不定义首页入口
- 左侧为固定浅色导航栏
- 左侧导航栏在视觉上需要更贴近页面左边缘
  - 不保留当前明显的栏外空隙感
  - 整体贴边关系参考 DeepTutor
  - 左栏需要直接贴住页面左边缘
  - 左栏需要贴住页面顶部和底部，不保留上下缩进
  - 左栏本体不使用圆角
- 左侧导航栏支持收起和展开
  - 默认保持展开
  - 收起后缩到 `72-88px`
  - 收起后保留品牌图标和一列主导航图标
  - 收起态默认显示：
    - 新话题图标
    - 必要的全局入口图标
  - 收起态不显示文字标签
  - 收起态不显示话题切换列表
  - 收起态图标采用纵向排列，信息密度贴近 DeepTutor
  - 不再保留没有真实功能承载的占位导航项
    - 当前会话
    - 历史记录
- 桌面端三栏采用接近参考图的非均分布局
  - 左栏固定 `248-256px`
  - 中栏外壳宽度控制在 `520-560px`
  - 中栏内部实际阅读列控制在 `540-600px`
  - 右栏默认展开态固定为 `560-620px`
  - 默认展开态优先保证右栏宽于中栏，不再采用“右栏占剩余空间”的松约束
- 左栏收起后的空间分配规则必须固定
  - 左栏收起后，中栏宽度保持不变
  - 左栏收起后，新增空间优先分配给右栏
  - 如果右栏本来已经收起，则继续保持右栏当前收起状态，不自动展开
- 中间为纯对话流主栏，必须是页面唯一主焦点
- 中间主栏最终只展示两类内容
  - user 发送的文字
  - agent 回复的文字
- 右侧为当前工作区，支持收起和展开，但视觉优先级必须低于中栏
- 右侧工作区默认更宽，但更宽不代表更强视觉权重
  - 右侧降权依靠更紧凑的信息密度
  - 右侧降权依靠默认折叠的分组
  - 右侧降权依靠更轻的表面层级和更小的字号层级
  - 不通过缩小右栏宽度来表达辅助角色
- 右侧工作区的收起状态必须由工作台根容器统一控制
  - 收起后中栏扩张
  - 收起后右栏缩到 `72-88px`
  - 收起动作在视觉上应向页面右侧收拢，而不是向中栏方向侵入
  - 收起态不保留 “Current Workspace / 当前工作区 / workspace” 等文字标识
- 左侧导航栏的收起状态也必须由工作台根容器统一控制
  - 左栏收起后只改变左栏与右栏的空间分配
  - 左栏收起后不允许中栏扩张
- 三栏都采用 `100vh` 内部滚动模型，不回退为整页长滚动
- 视觉风格高度贴近 DeepTutor，但不要求像素级复刻
- 全局视觉基线采用浅色底、弱边框、轻阴影、大圆角、中等留白
- 全局字号层级需要统一
  - 页面主标题 `28-32px`
  - 区块标题 `16-18px`
  - 卡片标题 `14-15px`
  - 正文 `14px`
  - 导航和按钮主文案 `13-14px`
  - 标签、状态、时间、元信息 `11-12px`
- 全局圆角层级需要统一
  - 小控件 `10-12px`
  - 普通卡片 `16px`
  - 主面板 `20-24px`
  - 左侧导航栏是例外，保持贴边直角
- 基础动效采用轻量过渡
  - 颜色反馈 `180-220ms`
  - 布局和透明度 `280-320ms`
- 中栏顶部信息需要减重
  - 话题标题保留，但不能继续放大
  - 话题说明和状态条必须缩成工具条级别
- 中栏输入区需要改成 DeepTutor 风格的轻量单行输入框
  - 采用单行输入框 + 发送图标按钮
  - 不再保留大块输入卡片
  - 输入区视觉权重必须明显低于中栏对话内容
- 页面仍需覆盖空、加载、成功、失败状态

## 验收标准

- 进入某个话题页面后显示三栏工作台
- 左侧可见导航、当前工作区和话题切换入口
- 左侧支持收起和展开
- 左侧视觉上贴近页面左边缘，不再保留明显栏外空隙感
- 左侧栏直接贴住页面左边缘、顶部和底部，不保留上下缩进
- 左侧栏本体不带圆角，和中栏、右栏的圆角面板形成区分
- 左侧收起后仍保留品牌图标和主导航图标列
- 左侧收起后不显示文字标签
- 左侧收起后不显示话题切换列表
- 左侧导航不再显示无真实承载价值的“当前会话”“历史记录”占位项
- 中间显示对话主栏，不再是纵向长页面分区
- 右侧显示当前工作区并支持收起/展开
- 桌面端右栏宽度明显大于中栏，整体接近参考图比例
- 桌面端默认展开态下，右栏宽度命中 `560-620px` 区间
- 桌面端默认展开态下，中栏外壳宽度命中 `520-560px` 区间
- 左栏收起后，中栏宽度保持不变，右栏宽度变大
- 如果右栏原本收起，左栏收起后右栏继续保持收起态
- 收起右侧工作区后，中栏会明显变宽，右栏会明显变窄
- 收起右侧工作区后，视觉上应向页面右侧收起，而不是向中栏方向折叠
- 右侧工作区收起态不再保留 “Current Workspace / 当前工作区 / workspace” 文案
- 左栏、中栏、右栏都可以独立滚动
- 打开页面后，视觉焦点首先落在中栏主栏，而不是右栏或顶部说明
- 右栏即使默认更宽，也不会压过中栏主焦点
- 中栏顶部信息和输入区的存在感明显低于主对话内容
- 中栏主栏整体表现为聊天产品，而不是日志/结果面板混排
- 输入区为单行输入框 + 发送图标按钮，而不是大块输入卡片
- 页面样式主要由 Tailwind utilities 驱动，而不是继续扩写全局样式文件
- 图标、按钮和容器级动效风格统一，符合已确认的 Tailwind + Lucide + Framer Motion 形式
- 全局标题、正文、导航、元信息的字号层级统一
- 面板、导航和按钮整体气质接近 DeepTutor 的轻量视觉
- 工作台允许采用“真实 API + 局部 mock”混合驱动
