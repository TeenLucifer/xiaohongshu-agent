# 001 Topic Workspace Tasks

## 开发子步骤

- [x] `001-A`：重定义三栏工作台级 Mock 数据模型
  覆盖当前话题、左侧导航、中间对话流和右侧工作区状态。
- [x] `001-B`：移除独立话题列表首页
  默认进入当前话题工作台，并把话题切换入口放入左侧栏。
- [x] `001-C`：实现三栏工作台骨架
  固定左侧导航、中间对话、右侧工作区的布局关系。
- [x] `001-D`：实现右侧工作区收起与展开
  保证三栏切换时布局稳定。
- [x] `001-E`：补齐新版工作台状态展示
  为主要区域补空、加载、成功、失败状态。
- [x] `001-F`：按参考图重调三栏宽度分配
  固定左栏宽度，压缩中栏宽度，放大右侧工作区宽度。
- [x] `001-G`：实现三栏独立滚动
  确保左栏、中栏、右栏都在视口高度内独立滚动。
- [x] `001-H`：修正右侧工作区收起联动
  收起状态改由工作台根容器控制，保证中栏扩张、右栏缩窄。
- [x] `001-I`：将三栏宽度比例调整到 DeepTutor 风格
  左栏固定 `256px`，中栏控制在 `420-460px`，右栏明显大于中栏。
- [x] `001-J`：接入全局基础视觉 token
  统一字号层级、圆角、阴影、边框和页面基础颜色。
- [x] `001-K`：为工作台壳层补齐基础动效
  覆盖三栏收起/展开、导航 hover 和常用按钮状态切换。
- [x] `001-L`：重构三栏主次关系
  让中栏成为唯一主焦点，右栏退为辅助上下文区。
- [x] `001-M`：调整中栏阅读列宽度
  将中栏收敛到 `560-680px` 的阅读友好范围。
- [x] `001-N`：压低中栏头部信息权重
  让话题说明和状态条缩成工具条级别。
- [x] `001-O`：将输入区改成轻量单行输入
  使用单行输入框 + 发送图标按钮替代大块输入卡片。
- [x] `001-P`：整体下调字号层级
  把页面标题、区块标题、正文、标签和按钮统一压到新的阅读节奏。
- [x] `001-Q`：接入 Tailwind CSS 并建立基础 token
  切到 utility-first 方案，只保留少量全局基础样式。
- [x] `001-R`：建立自定义轻量基础组件体系
  抽 Button、Input、Badge、Panel、Modal、ScrollArea、SidebarItem 等基础件。
- [x] `001-S`：接入 Lucide 图标并统一结构型图标用法
  只在导航、输入、开合和弹窗等结构型位置使用图标。
- [x] `001-T`：接入 Framer Motion 容器级动效
  仅覆盖壳层、分组、弹窗和页面级动效，不做满屏细碎动画。
- [x] `001-U`：调整左栏贴边关系
  让左栏视觉上更贴近页面左边缘，不再保留当前明显栏外空隙感。
- [x] `001-V`：重构左栏收起态内容模型
  收起后保留品牌图标和主导航图标列，不再只剩单个品牌图标。
- [x] `001-W`：固定左栏与右栏的独立收起逻辑
  左栏收起时不改写右栏当前状态，右栏已收起则继续保持收起。
- [x] `001-X`：隐藏左栏收起态的文字标签和话题切换
  收起态只保留纵向图标列，不显示文案和话题列表。
- [x] `001-Y`：移除左栏上下缩进与圆角
  让左栏贴满视口高度，顶部和底部不保留留白，并取消左栏本体圆角。
