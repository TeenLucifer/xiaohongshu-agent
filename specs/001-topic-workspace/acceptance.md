# 001 Topic Workspace Acceptance

## 手工验收

1. 打开前端应用并进入任意话题工作台
2. 确认页面为“左侧导航 + 单一工作区主面板”，不再存在独立右侧工作区列
3. 确认主工作区顶部可见 `选题 / 创作 / 对话` 三个 tab
4. 确认默认 tab 为 `选题`
5. 确认 `选题` tab 中可见素材、搜索结果，且总结作为搜索结果区下方的一部分呈现
6. 切到 `创作` tab，确认可见文案和图片，且图片区内有局部输入框
7. 切到 `对话` tab，确认可见消息流与输入框
8. 确认离开 `对话` tab 后输入框不再显示
9. 在 `选题` 或 `创作` tab 中点击会触发 agent 的按钮，确认页面保持停留在当前 tab
10. 再切到 `对话` tab，确认能看到刚才的 user / agent 执行记录
11. 收起左侧导航，确认主工作区自动扩展占满剩余宽度
12. 确认各 section 头部不再显示 `已就绪` 或 `进行中`
13. 在 `选题 / 创作` 的局部输入框发送一次消息，确认页面不切 tab，但 `对话` tab 中能看到对应记录
14. 确认页面整体仍保持浅色、轻量、内部滚动的工作台风格

## 自动化验收

- workspace-first 主布局测试通过
- `选题 / 创作 / 对话` tab 渲染与切换测试通过
- 对话输入框仅在 `对话` tab 中显示测试通过
- 左栏收起后主工作区扩展测试通过
- `选题 / 创作` 中触发 run 后停留当前 tab 的交互测试通过

## 已知限制

- 当前 tab 内内容组织继续复用现有分组组件和数据流
- 本轮只调整布局与交互心智，不新增新的后端协议
