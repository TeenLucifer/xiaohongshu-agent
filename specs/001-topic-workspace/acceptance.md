# 001 Topic Workspace Acceptance

## 手工验收

1. 打开前端应用
2. 进入任意一个话题工作台页面
3. 确认页面为左导航、中对话、右工作区三栏结构
4. 确认左栏宽度固定约 `248-256px`，中栏外壳宽度约 `520-560px`，右栏默认展开态宽度约 `560-620px`
5. 确认左侧栏直接贴住页面左边缘、顶部和底部，不再有当前明显的外侧或上下空隙感
6. 确认左侧栏本体不带圆角，和中栏、右栏的圆角面板形成清晰区分
7. 收起左侧栏，确认左栏缩到约 `72-88px`，并保留品牌图标和主导航图标列
8. 确认左栏收起态只显示图标，不显示文字标签
9. 确认左栏收起态不显示话题切换列表
10. 确认左栏收起后，中栏宽度保持不变，右栏宽度自动变大
11. 如果右栏原本处于收起态，收起左栏后确认右栏继续保持收起，不会自动展开
12. 收起右侧工作区，确认中栏会明显变宽，右栏会明显变窄
13. 确认右侧工作区在视觉上向页面右侧收起，而不是向中栏方向折叠
14. 确认右侧工作区收起后不再保留 “Current Workspace / 当前工作区 / workspace” 文案
15. 确认左侧导航中不再显示“当前会话”“历史记录”这类无实际承载的占位项
16. 确认左栏、中栏、右栏都可在视口内独立滚动
17. 确认右侧工作区可以收起和展开，且切换时有轻量过渡
18. 确认左侧栏中可见话题切换入口和必要的全局导航
19. 确认页面默认先看到的是中栏主内容，而不是右栏或顶部说明块
20. 确认右栏虽然默认更宽，但仍然通过更紧凑的信息密度和更轻的表面层级保持辅助角色
21. 确认中栏顶部信息比当前实现更轻，视觉权重明显低于主对话内容
22. 确认输入区已经改为单行输入框 + 发送图标按钮
23. 确认标题、正文、按钮、元信息的字号层级统一，并整体比当前实现更克制
24. 确认面板、导航和按钮整体风格接近 DeepTutor 的轻量视觉
25. 确认页面样式主要由 Tailwind utilities 驱动，而不是继续依赖大段全局样式
26. 确认图标只出现在导航、输入、开合、弹窗等结构型位置
27. 确认动效只出现在页面级和容器级，不存在满屏细碎动画
