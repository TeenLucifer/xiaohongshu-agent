# 003 Content Creation

## 背景

在新版三栏工作台里，模式总结、文案结果和图片结果不再作为独立长页面区块存在，而是拆分到中间对话流和右侧工作区中。它们的样式需要服从 `001` 中定义的全局视觉基线，并尽量贴近 DeepTutor 的信息密度。中间主栏只展示 user 与 agent 的对话结果，不展示过程性节点。

## 目标

- 在中间对话流中以 agent 结果消息展示模式总结和文案结果
- 让文案以对话节点形式进入主栏，但正式编辑入口固定在右侧工作区
- 在右侧工作区展示 Markdown 化的总结、可编辑的 Markdown 文案区和图片结果
- 在“图片”section 中同时承接编辑区与真实生成结果展示
- 提供右侧“总结/文案”生成按钮，复用现有 run 链路
- 让“创作”tab 中的图片编辑区与大图预览交互各自有独立能力边界

## 非目标

- 文案多版本对比
- 图片精修画布
- 复杂富文本编辑器

## 用户故事

- 作为运营人员，我希望中间主栏专注看 Agent 对话和结果节点，而不是再看到一堆独立大区块。
- 作为运营人员，我希望文案能在中间对话里被点开编辑，同时右侧还能看到当前文案和图片结果。
- 作为开发者，我希望先固定对话节点和右侧工作区的分工，再接真实生成结果。

## 输入输出

- 输入：真实 `pattern_summary.json`、真实 `copy_draft.json`、真实 `editor_images.json`、真实 `image_results.json`
- 输出：可查看总结、可编辑文案、可浏览右侧图片结果并消费聊天缩略图的三栏工作台状态

## 约束

- 中间主栏保持简化对话流优先
- 文案以 agent 结果消息形式出现，不常驻中间大编辑区
- 主界面不展示独立的结果摘要块
  - 总结结果不做摘要面板
  - 文案结果不做摘要面板
  - 都以 agent 回复消息直接呈现
- 右侧工作区采用双 tab 结构
  - “选题” tab 包含素材、搜索结果、总结
  - “创作” tab 包含文案、图片
- “选题” tab 中的总结区采用只读 Markdown 展示
- “选题” tab 中的总结区提供固定动作按钮
- “创作” tab 包含文案和图片两个分组，默认折叠
- “创作” tab 中的文案区采用可编辑的单文档 Markdown 展示，提供固定动作按钮
- 文案区编辑交互固定为：
  - 右侧只有一个文案编辑框
  - 输入 `#` / `##` / `-` / `>` 等 Markdown 语法后，当前行即时切换对应格式
  - `Enter` 提交当前行并开始下一行正文
  - `Shift + Enter` 在当前行中换行
  - 标题与正文编辑都需要自动写回 `copy_draft.json`
- “创作” tab 中的图片 section 由两部分组成：
  - 上半部分为 `007-image-editor-frontend` 承接的图片编辑区
  - 下半部分为 `025-image-generation-skill` 产出的图片结果展示区
- 图片结果区与图片编辑区中的大图预览统一由 `008-image-lightbox-preview` 承接
- 按钮本质上只是代用户发起一次 run，不新增第二套后端动作
- 按钮无论当前是否已有结果都保持可见，可用于重新生成和覆盖更新
- 图片结果区按单列表追加展示，不再按文生图和图生图分组
- 每张结果图都需要支持删除和拖拽进入编辑区
- final answer 下需支持展示 1 张代表图缩略结果
- 文案节点和右侧文案区都需要采用统一字号层级和按钮样式
- 右侧文案摘要和图片区需要与帖子结果区保持一致的圆角、边框和间距体系
- 中栏内容需要服从主阅读列节奏
  - 总结结果和文案结果应作为精简的 agent 回复内容出现
  - 右侧文案摘要和图片区必须明显弱于中栏主内容
  - 右侧即使位于更宽列宽中，也必须保持辅助角色
  - 右侧降权依靠默认折叠、更紧凑字号、更轻边框和更短摘要
  - 不通过缩小右栏宽度来表达辅助角色

## 验收标准

- 中间对话流能以 agent 回复消息展示模式总结结果
- 文案节点只读展示，提示用户到右侧继续编辑
- 右侧文案区支持单文档 Markdown 编辑，按 Enter 可即时提交当前行并开始下一行
- 右侧文案区编辑后会真实写回 `copy_draft.json`
- 右侧工作区包含”选题”和”创作”两个 tab
- “选题” tab 中能看到素材、搜索结果、总结摘要
- “创作” tab 中能看到文案摘要和图片结果
- “选题”/”创作” tab 都有固定动作按钮
- 图片结果在“创作” tab 的图片 section 下半部分展示
- 图片结果按单列表追加展示
- final answer 下可展示 1 张代表图缩略结果
- 右侧文案摘要和图片区与帖子区风格一致
- 文案摘要和图片区在更宽右栏下仍不抢中栏焦点
- 中栏总结结果和文案结果符合阅读列节奏，不显得过厚或过满
- 中栏不存在独立结果摘要块，结果内容直接以 agent 回复消息呈现
