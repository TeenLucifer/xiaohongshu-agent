# 003 Content Creation Acceptance

## 手工验收

1. 进入工作区主面板并切到 `对话` tab
2. 确认模式总结结果以对话节点形式出现
3. 确认文案节点在 `对话` tab 中只读展示，并提示到 `创作` tab 继续编辑
4. 确认文案节点与对话流整体字号、间距和视觉层级协调
5. 切到 `创作` tab，展开文案分组，确认可见可编辑的 Markdown 文案区
6. 在 `创作` tab 顶部展开搜索结果区里的总结折叠块，确认总结内容按 Markdown 只读展示
7. 确认 `创作` tab 中搜索结果区内存在“生成总结/重新生成总结”按钮
8. 确认 `创作` tab 中“文案”区存在生成按钮
9. 在 `创作` tab 展开图片 section，确认上半部分是编辑区，下半部分是生成结果展示区
10. 确认生成结果以单列表追加展示
11. 确认生成结果图支持删除与拖拽进入编辑区
12. 在文案区输入 `## ` 或 `- ` 等 Markdown 语法，确认当前块即时切换为对应格式
13. 输入较长文案直到超出可见高度，确认编辑器内部出现滚动条，并自动跟随当前编辑位置
14. 在文案区选中一段文本，确认出现“AI 润色”入口
15. 输入本次润色提示词并提交，确认只替换当前选区，不改其它内容
16. 确认润色完成后，`对话` tab 中新增一条结果消息
17. 刷新页面，确认修改后的文案仍能从 `copy_draft.json` 恢复
18. 确认 final answer 下可展示代表图缩略结果
19. 确认文案区和图片区与帖子区视觉风格一致
20. 确认 `对话` tab 中的总结结果和文案结果符合阅读列节奏，不显得过厚或过满
21. 确认 `创作` 中的搜索结果、总结、文案和图片区仍保持工作区式的辅助表达
22. 确认主界面不存在独立的结果摘要块
23. 在图片区局部输入框发送一条消息，确认页面保持在 `创作` tab，且 `对话` tab 中能看到对应记录
24. 确认总结内容为白板式直出，不出现明显的多层嵌套边框
25. 确认图片区局部输入框位于编辑区之后、生成结果区之前，且 placeholder 为 `输入你想继续让agent处理的内容...`
