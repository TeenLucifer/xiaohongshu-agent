# 006 Skills Page Acceptance

## 手工验收

1. 启动后端与前端
2. 点击左侧导航中的 `Skills`
3. 确认进入独立的 `/skills` 页面
4. 确认页面列出全部已加载 skills
5. 确认列表中能看到：
   - 名称
   - 描述
   - 可用状态
   - 来源位置
6. 若某个 skill 缺失依赖，确认其状态与缺失依赖提示可见
7. 点击任意一个 skill
8. 确认弹窗打开
9. 确认弹窗显示：
   - 名称
   - 描述
   - 来源类型
   - 文件位置
   - 可用状态
   - 缺失依赖
   - 正文摘要
10. 关闭弹窗，确认页面列表仍保持可交互
11. 返回首页或进入任意 topic，确认现有导航和工作台不受影响

## 自动化验收

- `GET /api/skills` 接口测试通过
- builtin、workspace、父 skill、子 skill 列表聚合测试通过
- 缺失依赖状态与 `requires` 字段测试通过
- 技能正文摘要去 frontmatter 测试通过
- `/skills` 路由渲染测试通过
- skills 列表页与详情弹窗交互测试通过
- 现有首页、工作台和候选帖页面测试通过

## 已知限制

- 第一版不提供搜索、筛选、排序、分组
- 第一版只读，不提供技能安装、启用或执行入口
- 正文展示为摘要，不追求完整 `SKILL.md` 阅读体验
