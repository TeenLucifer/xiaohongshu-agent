# 022 Streaming Run API Acceptance

## 手工验收

1. 启动已实现 `020` 的后端服务
2. 使用流式接口发起一次新的 run
3. 确认前端或调试客户端能收到 `run_started`
4. 在模型生成过程中，确认能持续收到文本增量事件
5. 当发生工具调用时，确认能收到工具调用开始事件
6. 工具执行完成后，确认能收到工具调用结束事件
7. run 正常结束时，确认能收到完成事件
8. 对比同步 `POST /runs` 的最终结果
9. 确认流式完成态与同步结果在核心字段上保持一致
10. 构造一次失败路径
11. 确认能收到失败事件，并且前端可识别失败结束
12. 流式完成后刷新消息列表
13. 确认最终消息历史与流式展示结果一致

## 自动化验收

- 流式事件协议测试通过
- `run_started` 事件测试通过
- 文本增量事件测试通过
- 工具调用开始/结束事件测试通过
- `run_completed` 事件测试通过
- `run_failed` 事件测试通过
- 同步/流式结果对齐测试通过
- 前端主栏流式消费测试通过
- 断流与关闭语义测试通过

## 已知限制

- 当前不覆盖完整 topic 真相层的流式更新
- 当前不引入复杂任务队列
- 当前不解决多 worker 分发问题
- 当前 `assistant_delta` 仍是最终文本的快速分段回放，不是 provider 原生 token 级流式
