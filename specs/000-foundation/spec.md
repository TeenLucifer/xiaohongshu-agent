# 000 Foundation

## 背景

建立一个可持续推进的 Python AI 项目基础工程骨架。

## 目标

- 配置层可用
- 日志层可用
- 异常层可用
- 最小测试链路可用

## 非目标

- 任何具体业务逻辑
- 复杂框架接入

## 用户故事
- 作为开发者，我希望新仓库从第一天起就有稳定的配置、日志、异常和测试基线。
- 作为协作者，我希望新功能能直接在这套工程骨架上推进，而不是先补脚手架。

## 输入输出
- 输入：环境变量、本地命令、测试运行上下文
- 输出：可用的配置对象、logger、基础异常类型和测试基线

## 约束
- 只做基础工程骨架，不引入业务特有模型或外部 API
- 配置层必须从环境变量读取，不允许在业务模块硬编码
- 质量门禁必须可直接运行

## 验收标准

- `uv run pytest -q` 可执行
- `uv run pyright` 可执行
- 基础配置对象能从环境变量加载
- 日志与异常模块有最小测试覆盖
