# 技术决策

## 工程基线

- Python `3.11+`
- `uv` 负责依赖和命令执行
- `ruff` 负责 lint 与格式检查
- `pyright` 负责静态类型检查
- `pytest` 负责测试

## 为什么这样选

### `uv`
- 命令统一
- 依赖解析和虚拟环境速度快
- 适合 AI 协作时减少环境摇摆

### `ruff`
- 一个工具覆盖大部分 lint 和格式需求
- 规则明确，适合保持代码一致性

### `pyright`
- 对类型问题反馈快
- 适合以 schema 和协议为中心的 Python AI 项目

### `pytest`
- 组织单元、集成、E2E 测试方便
- 适合严格 TDD 节奏

## 模板级默认约束

- 默认优先 `pydantic` schema，而不是跨层传裸 `dict`
- 默认先保留简单目录骨架，不预装重量级业务框架
- 默认先做同步、可调试实现，再评估是否需要异步或流式

## 需要显式记录的后续决策

新项目落地后，至少应补这些决策：

- 模型提供方与接口兼容层
- 状态编排是否需要框架
- 存储层和缓存层选型
- 日志和可观测性策略
- 部署方式与运行边界
